主要功能特色

智能文件問答

針對特定文件集提出問題，自動搜索相關內容並生成精確答案，支援PDF、Word、TXT等多種檔案格式。

多種檢索策略

支援SVM、TF-IDF、BM25、MMR等多種檢索方法，智能選擇最適合您問題的檢索策略，提升回答精準度。

檔案摘要生成

自動將冗長文件摘要成精簡內容，支援map-reduce和refine等摘要策略，幫助您快速掌握關鍵資訊。

模型評估與優化

內建問題集生成和模型評估功能，輕鬆測試不同模型和參數配置的效能，找出最適合您應用場景的設定。

代理模式功能

支援工具使用功能，讓語言模型可以呼叫維基百科搜索、文件儲存等外部工具，實現更複雜的任務處理流程。

靈活模型選擇

支援OpenAI、Azure OpenAI、Hugging Face、Google Gemini、Anthropic等多家大型語言模型，以及本地LLaMA-cpp模型。

快速開始

安裝

pip install akasha-terminal

基本使用

import akasha
import os

# 設置 API 金鑰
os.environ["OPENAI_API_KEY"] = "your_api_key"

# 初始化 Doc_QA 對象
ak = akasha.Doc_QA(search_type="auto")

# 進行文件問答
response = ak.get_response(
    doc_path="your_documents_folder/",
    prompt="你的問題",
)

print(response)

常見問題

Akasha 的主要特點是靈活性與全面性。它支援多種語言模型、嵌入模型和搜索策略，可以根據不同需求自由調整。同時，Akasha 不僅僅是 RAG 工具，還包含文件摘要、模型評估和代理功能等多種功能，能夠滿足企業和研究機構的多樣化需求。

選擇語言模型時需考慮幾個因素：預算、性能需求、隱私需求和部署環境。對於追求高性能且預算充足的情況，可選擇 OpenAI 的 GPT-4 或 Anthropic 的 Claude 3 Opus；如果資料隱私敏感或需要本地部署，可選擇 Hugging Face 模型或 LLaMA-cpp 本地模型；若需中文處理能力較強的模型，可優先考慮 Taiwan-LLaMa 或 ChatGLM 等模型。

Akasha 支援多種常見文件格式，包括：PDF、Word (DOCX)、純文本 (TXT)、Markdown (MD)、CSV 和 PowerPoint (PPTX) 等。系統會自動處理檔案格式，提取內容並進行分塊索引，無需額外轉換步驟。

資源需求主要取決於所選模型和文件量。使用雲端 API 如 OpenAI、Anthropic 時，本地資源需求較低，普通筆電即可。使用本地模型如 LLaMA-cpp 時，需要較強的 CPU 或 GPU 資源。大型文件集檢索時，至少需要 8GB RAM，建議 16GB 以上；使用本地大型語言模型時，建議配備 NVIDIA GPU 且 VRAM 至少 8GB。

Akasha 支援多語言處理，除了中文和英文外，還可處理日文、德文、法文等多種語言的文件。不過處理效果取決於所選的語言模型和嵌入模型對該語言的支援程度。對於特定語言的優化處理，可以選擇專為該語言訓練的嵌入模型，如日文可選擇支援日文的 Sentence Transformers 模型。
